## P3 异常详情

**事件**：第一次 master `kubectl set image` 时连续 3 次 `TLS handshake timeout`（API server 偶发抖动），导致命令未发出。slave rollout 顺序执行但脚本未察觉，**违反 skill "master 先 slave 再" 原则**。

**处置**：发现后立即补一次 master rollout，最终两 deployment 都达 v2026.06.16.3，期间零停机（slave 一直在服务流量）。

**改进**：后续 skill 应在每个 kubectl set image 后 `kubectl get deploy ... -o jsonpath` 立即校验 image 是否真的更新，而不是仅靠 `rollout status` 的 success（rollout status 对未变化的 deployment 也会返回 success）。
